webmeetInfra Documentation

webmeetInfra is the Ploinky repository for the WebMeet media runtime. It ships a single agent, liveKitServerAgent, which supervises Redis, Coturn, LiveKit Server, LiveKit Egress, and (in the prod profile) Nginx + Certbot inside one container.

The manifest pulls docker.io/assistos/livekit-server-agent:webmeet-infra by default. The image is published through the manual publish-livekit-server-agent.yml workflow in AssistOS-AI/container-image-builds, which checks out this repository as its build context, uses the centralized Dockerfile from container-image-builds/images/livekit-server-agent/Dockerfile, publishes linux/amd64 and linux/arm64 variants, includes Node 24 plus the Ploinky dependency-cache toolchain, and authenticates against Docker Hub using that repository's DOCKERHUB_TOKEN secret; the token value must never be stored in any repository.

Default and dev local profiles advertise the detected workstation IPv4 address as the LiveKit SFU ICE node address and publish browser-facing media/TURN ports on 0.0.0.0. This keeps Firefox on macOS/Podman from getting trapped on loopback-only ICE candidates while Redis, Egress health, and the supervisor health port remain loopback-only.

The DS specifications are the source of truth for contracts, runtime assumptions, and Ploinky invariants. Start with DS000-vision.md, DS001-coding-style.md, and DS002-livekit-server-agent.md before changing the consolidated runtime.
