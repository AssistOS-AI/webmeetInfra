liveKitServerAgent runs a pinned multi-architecture image and supervises Redis, loopback LiveKit signaling/API, LiveKit Egress, and private health. LiveKit advertises the topology-configured literal globally routable unicast public IPv4 and owns the single box UDP mux on 7882.

Public signaling is routed through the public Router service. Twirp administration is routed only through the private Router listener with authenticated policy and an exact current-generation caller assertion. Egress template/service 7980 and health 7981 are distinct, loopback-only, owner-checked listeners with semantic probes. The upstream wildcard-health binary is rejected; activation requires the published multi-architecture digest of the commit-pinned loopback patch.

Relay service is external. Ploinky brokers short-lived relay credentials to exact authorized consumers; this repository contains no relay daemon, public TLS proxy, certificate process, or long-term relay secret.
